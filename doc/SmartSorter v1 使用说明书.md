## 1. 快速开始

### 30 秒上手流程

1. **添加数据源** — 在「整理」页点击「选择文件夹」，添加要整理的目录
2. **新建规则方案** — 在规则配置区选择「＋ 新建方案」
3. **添加规则** — 点击「+ 添加规则」，配置 IF 条件和 THEN 动作
4. **分析预览** — 点击「🔍 分析预览」查看变更计划
5. **确认执行** — 检查无误后点击「▶ 执行」

> 所有文件操作在执行前都会先预览，不会直接修改文件。

## 3. 文件整理（核心功能）

### 3.1 添加数据源

在「整理」页顶部的**数据源**面板中操作：

- 点击 **「选择文件夹」** 打开系统目录选择对话框
- 可添加多个目录作为数据源
- 勾选 **「递归子目录」** 会扫描所有子文件夹中的文件
- 点击每条路径右侧的 ✕ 移除数据源

### 3.2 规则方案管理

规则方案是一组规则的集合，可保存和复用。

- **新建方案**：从下拉菜单选择「＋ 新建方案」
- **选择已有方案**：从下拉菜单中选择之前保存的方案
- **编辑方案名称**：直接修改顶部的方案名称输入框
- **保存方案**：点击 💾 按钮将当前编辑保存到本地

### 3.3 配置规则

每条规则由 **IF（条件）** 和 **THEN（动作）** 两部分组成。

#### IF — 条件配置

条件决定哪些文件会被规则处理。

**可选条件字段：**

| 字段        | 说明                   | 示例                |
| ----------- | ---------------------- | ------------------- |
| 文件名      | 不含扩展名的文件名     | `report`, `IMG_001` |
| 扩展名      | 文件扩展名（无点号）   | `jpg`, `pdf`, `mp4` |
| 完整文件名  | 含扩展名的完整名称     | `report.pdf`        |
| 文件大小(B) | 文件字节数             | `1048576`（= 1MB）  |
| 创建时间    | 文件创建日期           | `2024-01-01`        |
| 修改时间    | 文件最后修改日期       | `2024-06-15`        |
| 父目录      | 文件所在的直接父目录名 | `Downloads`         |

**可选操作符：**

- **文本类**：等于、不等于、包含、不包含、开头是、结尾是、正则匹配、属于、不属于
- **数值类**：大于、大于等于、小于、小于等于、介于、等于
- **时间类**：早于、晚于、最近N天

**逻辑组合：**

- **全部满足（AND）**：所有条件都满足时才匹配
- **任一满足（OR）**：任一条件满足即匹配

#### THEN — 动作配置

动作决定匹配的文件如何被处理。

**重命名（4 种模式）：**

| 模式     | 说明                   | 示例                                  |
| -------- | ---------------------- | ------------------------------------- |
| 查找替换 | 替换文件名中的指定文本 | `IMG_` → `照片_`                      |
| 添加前缀 | 在文件名前添加文本     | 前缀`2024_` → `2024_report.pdf`       |
| 添加后缀 | 在文件名后添加文本     | 后缀`_备份` → `report_备份.pdf`       |
| 序列号   | 按模板批量编号         | `{filename}_{seq}` → `report_001.pdf` |

**移动到：**

将文件移动到指定目标路径。支持使用[魔法变量](#7-魔法变量参考)动态生成路径。

- **目标路径**：如 `D:/Archive/{created_year}/{extension}`
- **冲突策略**：跳过 / 覆盖 / 自动重命名

**复制到：**

与「移动到」类似，但保留源文件。

**删除：**

删除匹配的文件。操作面板会显示 ⚠ 警告提示。

### 3.4 分析预览

点击 **「🔍 分析预览」** 后，系统会：

1. 扫描所有数据源中的文件
2. 逐一匹配规则条件
3. 计算目标路径 / 新文件名
4. 生成变更计划（不修改任何文件）

**预览面板说明：**

- **统计栏**：显示扫描数、匹配数、各操作类型数量、冲突数、错误数
- **变更列表**：
  - 红色文件名 = 原始名称
  - 绿色文件名 = 变更后名称
  - 每行显示操作类型、文件大小
  - ⚠ 标记表示存在冲突
- **复选框**：取消勾选可排除不想执行的文件

### 3.5 执行

确认预览无误后，点击 **「▶ 执行」**：

- 仅对勾选的文件执行操作
- 移动操作采用**安全三步法**：复制 → SHA-256 哈希校验 → 删除源文件
- 执行结果会自动写入历史日志
- 如有失败项，会提示成功/失败/跳过的数量

## 4. 重复文件检测与清理

在左侧导航点击「去重」进入。

### 使用步骤

1. **添加扫描目录** — 点击「选择文件夹」添加一个或多个目录
2. **开始扫描** — 点击「🔍 开始扫描」
3. **选择保留策略** — 使用快捷按钮或手动选择每组中要保留的文件
4. **执行删除** — 点击「🗑 删除 N 个重复文件」按钮清理

### 检测原理

采用**两阶段检测**确保准确性：

1. **文件大小预筛**：快速按文件大小分组，大小不同的文件不可能重复
2. **SHA-256 哈希精确比对**：对大小相同的文件计算完整哈希值

### 结果操作

- **统计摘要**：扫描文件数 / 重复组数 / 可释放空间
- **重复组卡片**：每组显示文件大小、副本数、哈希值前缀
- **选择保留**：
  - **单独选择**：点击每个文件前的 radio 按钮
  - **保留最新**：一键保留修改时间最近的文件
  - **保留最旧**：一键保留修改时间最早的文件
- 未被选为「保留」的文件会标记为「待删除」

### 执行删除

选定保留策略后，页面底部会出现 **「🗑 删除 N 个重复文件」** 按钮：

1. 点击删除按钮后，弹出**二次确认对话框**，显示即将删除的文件数量
2. 确认后开始删除，页面显示**进度条**和当前正在删除的文件路径
3. 删除完成后，以 **Toast 通知**显示成功删除和失败的数量

> ⚠ 删除操作不可撤销，请仔细确认后再执行。

## 5. 媒体智能归类

在左侧导航点击「归类」进入。

### 功能概述

对音频、视频、图片、电子书等媒体文件，基于文件名关键字和媒体元数据进行智能归类。系统会自动提取关键字，将同一关键字/作者的作品归入统一文件夹，并按 `关键字-主题.扩展名` 格式重命名。

### 使用步骤

#### 步骤 1：选择扫描目录

- 点击「选择文件夹」添加包含媒体文件的目录
- 默认**不递归**扫描子目录（可勾选「递归子目录」开启）

#### 步骤 2：配置分类参数

**媒体类型筛选：** 勾选要处理的文件类型

| 类型   | 涵盖格式                    |
| ------ | --------------------------- |
| 图片   | jpg, png, gif, webp, bmp 等 |
| 音频   | mp3, flac, wav, aac, ogg 等 |
| 视频   | mp4, mkv, avi, mov, wmv 等  |
| 电子书 | epub, pdf, cbz 等           |

**关键字来源：** 勾选要从哪些维度提取关键字

| 来源         | 说明                                 |
| ------------ | ------------------------------------ |
| 子文件夹名称 | 扫描目录下的直接子文件夹名作为关键字 |
| 作者/艺术家  | 从媒体文件的 Artist 标签提取         |
| 专辑艺术家   | 从媒体文件的 AlbumArtist 标签提取    |
| 专辑名       | 从媒体文件的 Album 标签提取          |
| 作曲家       | 从媒体文件的 Composer 标签提取       |

#### 步骤 3：开始扫描

点击「🔍 开始扫描」，系统会：

1. 扫描目录中的媒体文件
2. 从选定来源提取关键字
3. **自动合并包含关系的关键字**（如 "小凛蝶子" 包含 "蝶子"，合并为 "蝶子"）
4. 将文件与关键字进行匹配（文件名优先，元数据次之）

#### 步骤 4：处理多匹配文件

当一个文件匹配到多个关键字时，文件旁会显示**下拉选择框**，需要手动选择该文件应归入哪个关键字分组。

未匹配到任何关键字的文件会显示在「未匹配」分组中，这些文件将保持原位不动。

#### 步骤 5：预览与执行

1. 点击「📋 生成预览」查看每个文件的源路径 → 目标路径变更计划
2. 确认无误后点击「🚀 执行归类」
3. 系统执行移动与重命名操作，完成后**自动清理空的源文件夹**

### 命名规则

归类后的文件按以下格式命名：

```
{关键字}-{主题}.{扩展名}
```

- **关键字**：文件归属的关键字（如作者名）
- **主题**：从原文件名中去除关键字后的剩余部分
- **分隔符**：使用 `-` 连接

示例：

| 原文件名            | 关键字  | 归类后文件名         |
| ------------------- | ------- | -------------------- |
| `蝶子的作品001.mp4` | `蝶子`  | `蝶子-的作品001.mp4` |
| `作品A.mp4`         | `作者B` | `作者B-作品A.mp4`    |

## 6. 操作历史与撤销

在左侧导航点击「历史」进入。

### 日志信息

每条记录包含：

- **执行时间** 和 **耗时**
- **统计摘要**：总计 / 成功 / 失败 / 跳过
- **撤销状态**：可撤销 / 部分撤销 / 已过期
- **操作详情**：展开可查看每个文件的源路径、目标路径、操作类型

### 撤销操作

- 状态为「可撤销」的记录，点击 **「↩ 撤销」** 可还原文件
- **撤销原理**：
  - 移动 / 重命名：将文件移回原位
  - 复制：删除副本
  - 删除：**不可撤销**（请小心使用删除操作）
- 撤销后状态变为「已过期」，不可重复撤销

## 8. 魔法变量参考

在「移动到」「复制到」的目标路径模板以及重命名的前缀/后缀/序列号模板中，可使用以下变量：

| 变量               | 说明               | 示例值      |
| ------------------ | ------------------ | ----------- |
| `{filename}`       | 不含扩展名的文件名 | `报告`      |
| `{extension}`      | 扩展名（无点号）   | `pdf`       |
| `{full_name}`      | 完整文件名         | `报告.pdf`  |
| `{parent_dir}`     | 父目录名           | `Downloads` |
| `{created_year}`   | 创建年份           | `2024`      |
| `{created_month}`  | 创建月份           | `03`        |
| `{created_day}`    | 创建日期           | `23`        |
| `{modified_year}`  | 修改年份           | `2025`      |
| `{modified_month}` | 修改月份           | `12`        |
| `{modified_day}`   | 修改日期           | `01`        |
| `{size_mb}`        | 文件大小（MB）     | `2.5`       |
| `{seq}`            | 序列号（自动补零） | `001`       |

### 模板示例

| 场景         | 模板                                          | 效果                             |
| ------------ | --------------------------------------------- | -------------------------------- |
| 按年月归档   | `D:/Archive/{created_year}/{created_month}`   | `D:/Archive/2024/03/文件.pdf`    |
| 按类型分类   | `E:/分类/{extension}`                         | `E:/分类/pdf/文件.pdf`           |
| 按来源整理   | `F:/整理/{parent_dir}/{extension}`            | `F:/整理/Downloads/jpg/照片.jpg` |
| 批量编号     | `{filename}_{seq}`                            | `照片_001.jpg`, `照片_002.jpg`   |
| 添加日期前缀 | `{created_year}{created_month}{created_day}_` | `20240323_报告.pdf`              |

## 9. 常见问题

### Q: 文件操作安全吗？会不会丢失数据？

SmartSorter 采用多重安全机制：

- **预览优先**：所有操作在执行前必须经过预览确认
- **安全移动**：移动操作采用「复制 → SHA-256 校验 → 删除源文件」三步法
- **操作日志**：每次执行都会生成详细日志
- **一键撤销**：移动和重命名操作支持一键还原

### Q: 适合用于 NAS 网络存储吗？

适合。安全移动模式（复制 → 校验哈希 → 删除源）专为网络存储设计，确保数据传输完整性。

### Q: 支持多大的文件/目录？

- 目录扫描基于 `walkdir`，可处理数万级文件
- SHA-256 哈希采用 8KB 分块流式计算，不受文件大小限制
- 建议单次操作不超过 10,000 个文件以保持界面流畅

### Q: 规则方案保存在哪里？

规则方案和操作日志以 JSON 格式保存在系统应用数据目录：

- Windows: `%APPDATA%/com.smartsorter.app/`

### Q: 删除操作可以撤销吗？

不可以。删除操作一旦执行不可恢复。建议在包含删除动作的规则中仔细检查预览结果，确认无误再执行。

### Q: 正则表达式怎么用？

在条件配置中选择「正则匹配」操作符，值填标准正则表达式。例如：

- `IMG_\d{4}` — 匹配 IMG\_ 后跟 4 位数字
- `\.(jpe?g|png|gif)$` — 匹配图片扩展名
- `^[0-9]` — 匹配数字开头的文件名
