# 📁 SmartSorter v1.0 产品需求文档 (PRD)

**文档版本：** v1.0 (MVP)
**创建日期：** 2026-03-23
**目标平台：** Windows 桌面客户端
**技术栈选型：** Tauri (前端 Vue 3 构建 UI，后端 Rust 处理核心逻辑)

---

## 一、 产品概述

**SmartSorter** 是一款专为本地硬盘及 NAS 用户设计的桌面端文件智能整理工具。
针对现存整理软件“界面不友好、操作反人类、缺乏安全感”的痛点，SmartSorter 采用现代化的极简 UI，通过“可视化规则配置”与“整理前强制预览”机制，帮助用户安全、高效地完成海量文件的重命名、去重与自动归类。

---

## 二、 核心功能需求 (MVP 涵盖 6 大核心场景)

### 2.1 可视化规则配置 (Visual Rule Builder)

- **需求描述：** 提供零代码、所见即所得的规则配置面板，用户通过拖拽或下拉框组合整理逻辑。
- **功能细则：**
  - **条件块 (IF)：** 支持通过文件扩展名、包含的字符串、文件大小、创建/修改时间等条件进行组合筛选（如：当文件名包含 `截图` 且扩展名为 `png`）。
  - **执行块 (THEN)：** 支持设定重命名、移动、复制、删除等动作。
  - **规则复用：** 允许将配置好的规则集保存为“预设方案”（如：“清理桌面”、“归档NAS电影”），支持一键加载。

### 2.2 批量文件重命名 (Batch Rename)

- **需求描述：** 针对筛选出的文件，提供强大的批量重命名能力。
- **功能细则：**
  - **字符串操作：** 支持在原文件名中查找并替换、删除特定字符串。
  - **前后缀管理：** 统一在文件名前/后追加特定字符（如 `2023_`）。
  - **智能编号：** 支持按文件时间或大小排序后，自动追加递增序���（如 `001, 002...`）。

### 2.3 智能归类与路由 (Smart Routing)

- **需求描述：** 将选中的文件移动或复制到指定的现有文件夹，或根据规则自动新建文件夹。
- **功能细则：**
  - **动态目录生成：** 支持使用魔法变量拼接目标路径（例如：`Z:\归档\{扩展名}\{创建年份}\`）。
  - **自动创建目录：** 若设定的目标文件夹不存在，系统在执行时自动创建层级目录。
  - **冲突处理：** 移动/复制遇到同名文件时，支持选择“跳过”、“覆盖”或“自动重命名（追加序号）”。

### 2.4 重复文件检测 (Duplicate Finder)

- **需求描述：** 高效扫描指定目录，找出内容完全相同的冗余文件，释放存储空间。
- **功能细则：**
  - **双重校验算法：** 采用 `初筛（精确比对文件大小） -> 复核（计算 MD5/SHA-256 哈希值）` 的极速且安全的比对策略。
  - **可视化清理：** 将重复文件分组罗列，提供“保留最新”、“保留最旧”、“一键勾选其余副本”等快捷操作。

### 2.5 整理前预览 (Dry Run & Preview Tree)

- **需求描述：** 在实际对文件执行写操作之前，生成完整的预计效果图，提供极大的操作安全感（尤其针对 NAS 场景）。
- **功能细则：**
  - **虚拟执行：** 核心引擎仅在内存中计算规则结果，不触碰物理文件。
  - **Diff 差异视图：** 前端渲染清晰的树状图或列表对比。红色高亮显示“原文件状态”，绿色高亮显示“预计新状态（新名字/新路径）”。
  - **手工干预：** 用户可以在预览列表中手动取消勾选某个不想参与本次整理的文件。

### 2.6 日志与安全撤销 (Logs & Undo)

- **需求描述：** 记录每次整理操作的详细流水，并提供后悔药机制。
- **功能细则：**
  - **操作快照：** 每次执行完毕后，系统记录所有处理文件的映射关系表（`[原绝对路径] -> [新绝对路径]`）。
  - **一键撤销：** 用户在“历史记录”中点击撤销，系统根据映射表将文件原路退回，恢复原名（前提是原文件/原目录未被二次破坏）。
  - **执行日志：** 记录每次任务的耗时、成功数量、失败原因（如权限不足），方便排查。

## 三、 业务操作流程 (User Journey)

1. **导入数据：** 用户将待整理的文件夹（本地或 NAS 路径）拖拽至软件主界面。
2. **配置/选择规则：** 在面板中添加“重命名”、“路径路由”等规则，或直接选择已保存的“预设方案”��
3. **分析与预览：** 点击【分析】按钮，系统极速扫描并生成 Diff 预览树。
4. **人工核对：** 用户查看预览树，确认重命名和路径归类是否符合预期，微调勾选项。
5. **物理执行：** 点击【开始整理】，弹出进度条，后端多线程执行 I/O 操作。
6. **记录落盘：** 完成后生成本次操作的日志快照，支持后续一键撤销。

## 四、 非功能性需求与技术要求

1. **UI/UX 体验：** 界面需符合现代化审美，摒弃极客式的复杂表单。所有耗时操作必须有清晰的进度条与状态提示，不可产生“假死”现象（由 Vue 3 异步渲染保障）。
2. **性能要求：** 面对 NAS 上超过 10,000+ 个文件的目录结构，扫描及预览生成时间应控制在秒级（由 Rust `walkdir` 保障）。
3. **网络容错安全性：** 对 NAS 文件的跨目录移动，需采用 `复制 -> 校验 -> 删除原文件` 的安全策略，防止因局域网抖动导致文件损坏或丢失。
4. **资源占用：** 处于后台或待机状态时，内存占用应保持在极低水平（得益于 Tauri 架构）。
